    Helm and Kustomize

    This tutorial introduces Helm and Kustomize, two powerful Kubernetes configuration management tools. You'll learn the basics of each tool through practical exercises and see how combining them can simplify and enhance your Kubernetes deployments.

    The code examples and instructions in this tutorial are located under openshift-quickstart project in the tutorials/helm+kustomize directory. Ensure you are in this directory before executing the commands.

    1. Navigate to the Tutorial Directory 

      # Change to the tutorials/simple directory
cd openshift-quickstart/tutorials/helm+kustomize

    2. Or open a New Terminal

    Exercise 1: Helm Basics

    Objective:

    Deploy a simple application using Helm.

    Steps:

    1. Create a Helm Chart 
      helm create myapp
    2. Customize the Chart
    3. Edit values.yaml to customize your deployment: 
      image:
  repository: registry.redhat.io/ubi9/nginx-122
  tag: latest
    4. Deploy the Chart 
      helm install myapp-release ./myapp
    5. Verify Deployment 
      oc get pods

    Exercise 2: Kustomize Basics

    Objective:

    Deploy the same application using Kustomize.

    Steps:

    1. Create Base Manifests
    2. Create a directory structure: 
      myapp-kustomize/
├── base/
│   ├── deployment.yaml
│   └── service.yaml
└── overlays/
    └── dev/
        └── kustomization.yaml
    3. Define Base Resources
    4. Populate deployment.yaml and service.yaml with basic Kubernetes manifests.
    5. Create an Overlay
    6. In overlays/dev/kustomization.yaml, reference the base and apply customizations: 
      apiVersion: kustomize.config.k8s.io/v1beta1
kind: Kustomization
resources:
  - ../../base
patchesStrategicMerge:
  - deployment-patch.yaml
    7. Create deployment-patch.yaml to modify replicas or image tags.
    8. Deploy with Kustomize 
      oc apply -k overlays/dev
    9. Verify Deployment 
      oc get pods

    Exercise 3: Combining Helm and Kustomize (Overlay Power)

    Objective:

    Demonstrate how Kustomize overlays can enhance Helm deployments.

    Steps:

    1. Render Helm Templates
    2. Generate Kubernetes manifests from your Helm chart: 
      helm template myapp ./myapp > base.yaml
    3. Setup Kustomize Overlay
    4. Create a new directory structure: 
      helm-kustomize/
├── base/
│   └── base.yaml
└── overlays/
    └── staging/
        ├── kustomization.yaml
        └── patch.yaml
    5. In overlays/staging/kustomization.yaml: 
      apiVersion: kustomize.config.k8s.io/v1beta1
kind: Kustomization
resources:
  - ../../base/base.yaml
patchesStrategicMerge:
  - patch.yaml
    6. In patch.yaml, apply environment-specific changes (e.g., resource limits, replica counts): 
      apiVersion: apps/v1
kind: Deployment
metadata:
  name: myapp
spec:
  replicas: 3
  template:
    spec:
      containers:
        - name: myapp
          resources:
            limits:
              memory: "256Mi"
              cpu: "500m"
    7. Deploy Combined Configuration 
      oc apply -k overlays/staging
    8. Verify Deployment 
      oc get pods
oc describe deployment myapp

    Exercise 4: Using Kustomize with Helm Charts (--enable-helm)

    Objective:

    Leverage Kustomize's built-in Helm integration to directly manage Helm charts without manually rendering templates.

    Steps:

    1. Create Directory Structure 

      mkdir -p helm-kustomize-direct/overlays/prod

    2. Define Kustomization with Helm Chart Create a kustomization.yaml file in helm-kustomize-direct/overlays/prod: 

      apiVersion: kustomize.config.k8s.io/v1beta1
kind: Kustomization

helmCharts:
  - name: myapp
    repo: file://../../myapp
    releaseName: myapp-prod
    namespace: prod
    valuesInline:
      replicaCount: 4
      image:
        repository: myapp-image
        tag: "v2.0.0"

    Note: Adjust the repo path to point correctly to your local Helm chart directory.

    1. Deploy Using Kustomize with Helm Enabled 

      oc apply -k helm-kustomize-direct/overlays/prod --enable-helm

    2. Verify Deployment 

      oc get pods -n prod
oc describe deployment myapp-prod -n prod

    Explanation:

    • The --enable-helm flag allows Kustomize to directly process Helm charts, simplifying the workflow by eliminating the need to manually run helm template.
    • Inline values (valuesInline) provide a convenient way to customize Helm chart parameters directly within the Kustomize configuration.
